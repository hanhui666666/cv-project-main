## 2. 核心模块设计

### 2.1 模型模块 (`models/`)
| 文件 | 功能 | 关键参数 |
|-----|------|---------|
| `cbam.py` | CBAM注意力模块 | reduction_ratio, kernel_size |
| `eca.py` | ECA注意力模块 | kernel_size |
| `bifpn.py` | BiFPN特征金字塔 | num_channels, num_repeats |
| `loss/focal_loss.py` | Focal Loss | alpha, gamma |
| `loss/wiou.py` | WIoU损失 | - |

### 2.2 配置模块 (`configs/`)
每个实验方向对应一个YAML配置文件，包含：
- 模型配置（预训练权重、改进开关）
- 训练超参数（epochs、batch size、学习率）
- 数据配置（数据集路径、图像尺寸）
- 输出配置（项目路径、实验名称）

### 2.3 脚本模块 (`scripts/`)
| 文件 | 功能 | 使用方式 |
|-----|------|---------|
| `train.py` | 统一训练入口 | `python scripts/train.py --config xxx.yaml` |
| `eval.py` | 统一评估入口 | `python scripts/eval.py --config xxx.yaml --weights xxx.pt` |
| `infer.py` | 推理预测入口 | `python scripts/infer.py --source xxx.jpg` |
| `gradcam_vis.py` | Grad-CAM可视化 | `python scripts/gradcam_vis.py --source xxx.jpg` |
| `cross_domain.py` | 跨域迁移脚本 | `python scripts/cross_domain.py --config xxx.yaml` |

### 2.4 工具模块 (`utils/`)
| 文件 | 功能 |
|-----|------|
| `dataset_utils.py` | 数据集划分、格式转换、结构检查 |
| `metrics.py` | IoU、AP、mAP、混淆矩阵等指标计算 |
| `visualizer.py` | 边界框绘制、训练曲线、混淆矩阵热力图 |
| `logger.py` | 日志记录、实验管理、计时器 |

## 3. 关键技术实现

### 3.1 注意力模块集成
CBAM/ECA模块的集成位置：
- Backbone的C2f模块之后
- 可通过配置文件开关控制
- 不影响原始模型加载

### 3.2 损失函数替换
- 分类损失：替换为Focal Loss处理类别不平衡
- 回归损失：替换为WIoU提升定位精度
- 通过配置文件动态选择损失函数

### 3.3 Grad-CAM实现
- 目标层：选择模型最后一层卷积特征图
- 支持方法：GradCAM、GradCAM++、HiResCAM
- 输出：热力图、叠加图、对比图
- 批量处理：支持文件夹级别的批量生成

## 4. 训练流程

```
加载配置文件
    ↓
初始化模型（加载预训练权重）
    ↓
应用改进模块（注意力/损失/BiFPN等）
    ↓
数据加载与增强
    ↓
训练循环：
    ├─ 前向传播
    ├─ 损失计算
    ├─ 反向传播
    ├─ 参数更新
    └─ 验证与日志记录
    ↓
保存最优模型
    ↓
生成实验报告
```

## 5. 实验设计

### 5.1 对比实验
所有改进方向统一以下控制变量：
- 相同的训练/验证数据集
- 相同的训练轮次（100 epochs）
- 相同的图像尺寸（640×640）
- 相同的优化器（SGD）和学习率策略

### 5.2 评价指标
| 指标 | 说明 |
|-----|------|
| mAP50 | IoU=0.5时的平均精度 |
| mAP50-95 | IoU=0.5~0.95的平均精度 |
| Precision | 精确率 |
| Recall | 召回率 |
| FPS | 推理速度 |
| Params | 模型参数量 |
